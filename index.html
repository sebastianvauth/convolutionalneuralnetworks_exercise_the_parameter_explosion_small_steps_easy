Exercise: The Parameter Explosion

We've learned that traditional neural networks struggle with images. It's because images have lots of pixels, and in a fully-connected network, every pixel connects to every neuron in the next layer. That's a ton of connections! Each connection has a 'weight,' which is a number the network needs to learn. Let's do some simple math to see how quickly this adds up, even for a tiny image.

Continue

Imagine a grayscale image that's only 28 pixels wide and 28 pixels high. Think of a super-blurry, almost unrecognizable picture! We'll also use a small hidden layer with just 100 neurons.

Our first job is to figure out how many input 'nodes' we have. Each pixel is one input node.

Continue

For our 28x28 grayscale image, it's easy: we just multiply the width by the height.

\[ \text{Input Nodes} = \text{Width} \times \text{Height} = 28 \times 28 = 784 \]

Continue

Now, the crucial part: each of those 784 input nodes connects to every single one of the 100 neurons in our hidden layer. So, for each neuron in the hidden layer, there are 784 weights. To get the total number of weights, we multiply:

\[ \text{Weights} = \text{Input Nodes} \times \text{Hidden Neurons} = 784 \times 100 = 78,400 \]

Continue

We're not quite done! Each neuron in the hidden layer also has a 'bias' value. Since we have 100 neurons in the hidden layer, we have 100 biases.

\[ \text{Biases} = \text{Number of Hidden Neurons} = 100 \]

Continue

Finally, to find the total number of parameters the network needs to learn, we just add the weights and the biases:

\[ \text{Total Parameters} = \text{Weights} + \text{Biases} = 78,400 + 100 = 78,500 \]

Continue

So, even with this tiny image and a small hidden layer, we already have 78,500 parameters! That's a lot of numbers for the network to adjust during training. This is a first glimpse of the 'parameter explosion' problem.
